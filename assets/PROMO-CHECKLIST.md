# ai-notify プロモ動画 チェックリスト（宇野に出す前に必ず3回チェック）

> 宇野に同じ指摘を二度させない。動画を出力したら**毎回この全項目を自分で3パス目視**してから渡す。
> 「数値が出た＝OK」ではない。**出荷した実 mp4 のフレームと音を実際に確認**する。

## 1. 文字化け（グリフ）
- [ ] Hiragino にグリフが無い記号は**豆腐(□)化**する。使用禁止/要確認：
  - `−`（U+2212 マイナス記号）→ ASCII `-` を使う
  - 絵文字（🔊🔔✅🚀⭐ 等）→ カード/字幕では使わない（端末内の絵文字は実機由来なのでOK）
  - 曲線引用符 `“ ”`（U+201C/D）→ ASCII `"` を使う
  - 矢印・特殊記号（→ ・ — · ★ 等）は**使う前に出力フレームを必ず目視**。tofu なら ASCII へ。
- [ ] 字幕・カードの**全テキストをフレーム抽出して目視**（推測でOKにしない）。

## 2. 字幕と音声の同期
- [ ] **全ての発話に字幕を付ける**（一部だけ字幕にしない＝抜けた所で「ズレてる」に見える）。
- [ ] 各字幕の表示時刻 = 対応音声の発話時刻（onset一致）。次の発話開始で切り替える（同時に2枚出さない）。
- [ ] 字幕テキスト = その音声が言っている内容の訳になっているか（順番ズレ厳禁）。
- [ ] grid を post で配置する場合、overlay の `enable='between(t,…)'` の `t` は**入力ローカル時刻**。音声は最終タイムラインの絶対時刻。両者の基準(offset)を合わせる。

## 3. バイリンガル / コピー
- [ ] **日本語キャッチコピーを消さない**。カードは日英併記（JP大＋EN小）。
- [ ] 特にアウトロ：日本語タグライン「音・読み上げ・通知で、AIエージェントの『今』を逃さない」を必ず残す。
- [ ] 訴求から外す指定（例：ツンデレ）が確実に映像・字幕から消えているか。

## 4. 出荷ファイルの検証（元データでなく実 mp4）
- [ ] `ffprobe` で**音声ストリームが在る**こと（`-show_entries stream=codec_type`）。
- [ ] 声のレベルが本編で出ている（無音 -91dB でない／BGMに埋もれていない＝声がBGMより十分上）。
- [ ] BGM がうるさすぎない（声より十分下、目安 -15dB 以上下）。
- [ ] 問題報告のあった時刻（例 16s/24s/27s）を**ピンポイントでフレーム抽出して再確認**。

## 5. フォント・名称（必須）
- [ ] **フォントは Noto Sans 系**（`~/Library/Fonts/NotoSansCJKjp-Regular.otf` / `-Bold.otf`）。Hiragino 等に戻さない。
- [ ] **製品名は全部小文字「ai-notify」が正式**。`AI NOTIFY` / `AI-notify` / `ainotify`(ハイフン無) / 大文字頭は**一切禁止**。出力前に `grep -iE "ai[- ]?notify"` で全表記を確認。
- [ ] 「AI」単体（AIエージェント等の一般語）は可。禁止は製品名としての大文字/ハイフン無し表記のみ。

## 6. 運用
- [ ] `x-promo.mp4`（正本）を**勝手に上書きしない**。assets に見覚えない成果物があれば中身確認してから。
- [ ] 上記1〜4を**3回**通してから宇野に渡す。1回でも飛ばさない。
